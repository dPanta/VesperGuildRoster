## Implemented Changes

| Area | Status | Notes |
| --- | --- | --- |
| TOC | Updated | `vesperTools.toc` now targets `120005`. |
| Mage travel buttons | Updated | Removed direct `C_Spell.CastSpell` / `CastSpellByID` use from menu callbacks. Mage buttons are now secure action buttons: left-click casts the selected spell, right-click selects portal/teleport. |
| Spellbook scan | Updated | Portal mage spell discovery now uses `C_SpellBook.GetNumSpellBookSkillLines`, `GetSpellBookSkillLineInfo`, and `GetSpellBookItemInfo`. Legacy global spellbook scan was removed from the retail path. |
| Spell cooldowns | Updated | `C_Spell.GetSpellCooldownDuration` is called with `ignoreGCD=true` and wrapped with `pcall`. |
| Talent loadout search result | Updated | Removed nonexistent `C_ClassTalents.SetActiveConfigID` path. Search activation now tries `C_ClassTalents.SwitchToLoadoutByIndex`, `ClassTalentHelper.SwitchToLoadoutByIndex`, then `C_ClassTalents.LoadConfig(configID, true)`. |
| Bank / container APIs | Reviewed | Existing live-session and combat guards remain in place for `C_Bank.AutoDepositItemsIntoBank`, `C_Bank.IsItemAllowedInBankType`, `C_Container.GetContainerItemInfo`, and item pickup paths. In-game smoke testing still required for taint. |
| Unit APIs | Reviewed | Current callsites use public tokens such as `player`, `partyN`, and `raidN`; no GUID-to-secret-token pattern was found. |
